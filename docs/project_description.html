Project DesriptionProject

Topic

We envision creating a Discord bot that can run (at first) Python code, for use in Computer Science and IT related Discord servers or even private servers for hobbyist use, so that people can collaboratively view python outputs for any commands that are run and also display code that is pulled from a GitHub repository. It will act almost like a pseudo-version of the Python Integrated Development and Learning Environment (IDLE), so that everyone can view the code output at the same time, without the unnecessary bandwidth used by screensharing programs. Upon setting up the bot to run Python, other coding languages will then be integrated to be used with the bot. Examples including, JavaScript, C++, C#, Flash and Swift.


The bot will be hosted by an out scoured virtual private sever to ensure a 24/7 up time allowing for people from all around the world in different time zones to have access to the bot and its complete functionality. By hosting the bot on a VPS we also envision it being much cheaper and more efficient to run than if our team were to host it ourself which would most likely use a greater amount of resources and be more costly than if we were to simply pay a service such as Amazon Web Services to host it for us. Upon successful completion of the bot we hope to allow more people to have access to python through discord and increase the amount of people learning and collaborating with their code.

Motivation

Our motivation for this project comes from doing Foundations of Computing at the University of Melbourne, where we would constantly be discussing various workshop problems in the subject’s Discord server. We’d always argue about correct solutions, because people either had typos or syntax errors, or the version of Python they were running had quirks that weren’t present in the version of Python we were recommended to run. A bot like this would have saved us all a lot of time, as we could collaborate on the code in the server, instead of working on it individually. Also, with the addition of using Trinket during ‘Introduction to programming’. Many users who are doing Bachelor of IT know how buggy it Is when trying to write a program and then trying to save it. This provides further reason of why implementing python bot on to Discord would be a good idea.


Landscape

Upon researching information to create our bot, we have found that two competing discord bots already exist and perform vaguely similar functions to the bot we wish to create but both are lacking in some departments. The two bots being RTFM (created by a discontinued account) which most similarly mirrors our project idea and Python (created by Retox). When reviewing RTFM, we found that it performs the main function of running various coding languages including Python within the discord server, however, it was very hard to navigate and there was little to no help or guide to begin coding using the bot after adding it to the sever. RTFM was also running on an outdated version of Python which is almost 6 months behind and does not provide the option to update or select a Python version to use, nor is there any information on future patches. Furthermore, RTFM does not actually integrate coding within its bot but rather takes the input from the user, copies it to the website “https://tio.run/#”, where it is run and then the ouput is copied by the bot as an output for the user.


“Python” the Bot only has one primary function, being to run a Python Script from a Github Repository as well as another repository called pypi.org. The Bot once again offers little to no help and guidance upon joining a sever and takes a bit of guess work to get code to run within discord. It also does not allow the user to type and run Python code within the Discord server itself, as such we do not view Python the Bot as a direct competitor of our project idea.


Taking into consideration the functions and flaws of the two competing bots RTFM and Python, we envision creating a bot that has the core functionalities of both of the bots in a single entity with increased user friendliness and guidance when first inviting the bot into a sever. This will be done through a welcome message in the server’s designated main text channel, saying “Hello world!” and giving a list of commands that users can run including “help” which will list different areas in which the user might want more information on the functionalities of the bot. Furthermore, by integrating coding into the bot itself, the bot would not be reliant on an external website such as “https://tio.run/#” being up in order to process code and would only be reliant on the uptime of its host, in our case Amazon Web Services which has reliable backup severs in case of failures.

Detailed Description


Aim

The aim of the project is to create a Discord bot that runs Python code hosted by a virtual sever (hosted by Amazon Web Services). This would hopefully make Python more accessible to people who can’t install programs on their computer whether it be due to hardware requirements or any issues that have arisen from the including machines that are used in public spaces as Discord is able to run in most web browsers and on mobile devices. Hosting the bot off a virtual server would also ensure that the bot would run reliably with a 24/7 uptime and is capable of processing commands from multiple servers at once. This bot will attempt to fill a space for hobbyist and professional programmers to share and collaboratively work on Python in a more accessible environment like a discord sever, removing any system differences and issues.


Goals

  • Write up a bot in python that can run Python code inputs and produce outputs similar to how it would perform in an IDLE.
    • As the aim of the entire project is to create a bot that can run python, this goal is the primary function of the entire bot. If this goal isn’t achieved first and foremostly the entire project would not be able to progress any further. The main point of Python running in an IDLE is so that beginner programmers as well as professional ones will be able to efficiently and effectively write code similarly to how they would outside of Discord as the format would be the same in any other IDE (integrated development environment.

  • Ensure that the bot runs properly on a local system when hosted on a local network and system.
    • This goal ensures that the bot behaves as intended when hosted and deployed in a discord sever. Local host testing allows us to make sure that the bot can read commands and inputs and produce proper outputs without any errors before it is deployed on a virtual sever for hosting. This also allows for any changes and bug fixes to be made can be done so quickly as the bot itself is stored locally on someone’s computer.

  • Deploy the bot on cloud infrastructure (most likely amazon web services)
    • This goal is the second primary step of the entire project. By hosting the bot, we will be able to utilise sever machines much stronger than any of hardware and allow for constant uptime as it would be very unfeasible and costly to keep a locally hosted system on for days at a time. This would also potentially allow the bot to run faster with a dedicated system supporting it.

  • Test the bot when hosted by virtual severs.
    • This step is simply to ensure that the bot behaves as expected when hosted from a virtual sever and is a semi-final step to allow for any bugs or issues that may arise when the bot is sever hosted and not locally hosted. Only one running iteration of the bot in a single discord sever will be tested to ensure that all key functionalities perform as expected or similarly to how the bot was performing when hosted by a local sever.

  • Add the bot to multiple severs at once and stress test the bot.
    • If the bot is able to successfully be deployed in multiple severs at once and run effectively and efficiently then we can then begin stress testing and seeing how well our virtual private sever can handle running multiple instances of python code at a time and returning outputs to several different locations. This is to test how powerful the sever and service we have chosen is and if it is able to effectively handle the amount of demand that could potentially be observed by users in the future.

  • Add Java, C and other code integration
    • Once the bot is able to successfully run python in multiple different severs without any downtime or bugs then we can begin to program and add other coding languages and the commands to use them in the bot. Adding other languages to the bot will hopefully allow for more users of the bot and could even allow for potentially coding with multiple integrated languages for a single program. This goal could be ongoing for any additional programming languages that wish to be added by the community as well as to update existing codes as future versions are released.

  • Add GitHub functionality
    • Adding the functionality for GitHub will be the last functionality goal which will allow for users to pull code from GitHub repositories and seamless run them through their bot on discord severs. This will hopefully allow for multiple users working on projects to quickly be able to demonstrate their code without going through the hassle of either screensharing themselves running the code on their own IDEs or by each member of the group individually downloading files from the repository to then run on their own simultaneously.

Scope and Limits

In terms of our scope and limit for this project we plan to create a simple discord bot that is able to run Python code when hosted locally on our own hardware and computers. The bot should be able to registers inputs and produce outputs similar to how it would on an IDLE. The bot would be activated through the use of a “!command” followed by a string of code written in the discord chat box which should adhere to standard python string format. After hitting “enter” and sending the code through to the bot, the bot should then be able to output a result that reflects the code inputted by the user. If the bot is unable to read and process the code, it will relay an error message and where the bot ran into the error. For example, if the user had forgotten to define a variable in their input, the bot would relay “SyntaxError: invalid syntax”, and the line in which it had occurred.

Due to time constraints and difficulties in programming the bot, particularity in the bot taking considerably more time to get the code working due to us having to learn programming techniques and functions that were not taught to us in our first year programming subject. We have discovered that our limits in time constraints have led us to forgo many of our other key features of our bot, such as hosting it on VPS in this case Amazon Web Services. As this was one of the key features of our bot we would have liked to demonstrate this in our pitch by creating a fresh discord account and sever on a random computer at RMIT libraries and adding the bot to the sever and then demonstrating some code. We have also had to forego implementing the GitHub functionalities, once again due to time constraints, thus we will not be able to demonstrate pulling any code from GitHub repositories in our pitch video either. Finally, our last goal of wanting to implement other coding languages into the bot will have to be left out as we had unforeseen difficulties attempting to code an operational bot in Python alone, it would not be feasible to implement another coding language with our current skills and capabilities at present.
